# Contributing

Brain Hub accepts changes that preserve three invariants: source events stay immutable, graph claims remain traceable to evidence, and capture never breaks the host agent.

## Development loop

1. Add or update a test that demonstrates the behavior.
2. Make the smallest compatible schema or API change.
3. Run `make test` and `make lint`.
4. If the event contract changes, add a new versioned event type; never reinterpret an existing payload in place.
5. If an extractor changes, increment its version and prove a full replay is deterministic.

Do not add transcript or private database parsers unless the host publishes a compatibility contract. New adapters must use the generic manifest, deterministic event IDs, a bounded local spool, short timeouts, and metadata/summary capture by default.

`make install` performs a non-editable local wheel install because hardened Python runtimes may ignore the hidden `.pth` files generated by editable build backends. Re-run it after changing Python package code before testing installed console scripts.
